SECURITY DETAILS

ALGORITHM

  • Cipher: Serpent-256-CBC with PKCS7 padding
  • Integrity: HMAC-SHA256 — covers header + ciphertext, verified before decryption
  • Key derivation: Argon2id — 19 MiB RAM, 2 passes, 1 thread (OWASP 2023 minimum)
  • IV: 128-bit, unique per encryption, generated by window.crypto.getRandomValues

WHAT THIS PROTECTS

  • Confidentiality — ciphertext reveals nothing about the plaintext without the key
  • Integrity — any modification to the ciphertext or header is detected before decryption
  • Authentication — "authentication failed" means wrong key OR tampered data (intentionally indistinct)

WHAT THIS DOES NOT PROTECT

  • Metadata — file names, sizes, and the fact that you used this tool are not hidden
  • Traffic analysis — this tool only encrypts; it does not anonymize your network activity
  • Endpoint compromise — if your device is compromised, encryption cannot help
  • Weak passphrases — Argon2id is memory-hard and GPU-resistant, but cannot compensate for a weak passphrase

IMPLEMENTATION NOTES

  • All operations are local — nothing leaves the browser, no network calls
  • HMAC comparison uses constantTimeEqual — timing oracles are prevented
  • Argon2id runs via embedded WebAssembly — memory-hard, GPU/ASIC-resistant
  • Serpent-CBC and HMAC-SHA256 use leviathan-crypto (WASM-backed)
  • Argon2id uses the argon2id npm package (WASM-backed)
  • Limitation: files are processed entirely in memory — not suitable for very large files
